## Findings considered and rejected

From the 2026-07-02 deep audit (so they aren't re-reported next run):

- **"viem 2.53+ requires TypeScript 5.9"** — false; viem 2.54.0's peer dep is `typescript >=5.0.4` (checked node_modules/viem/package.json).
- **"Redundant access lists between candidate discovery and slot probing"** — misread; discovery access-lists the user's calls, probes access-list `balanceOf`/`allowance` calldata. Different queries, no redundancy.
- **ERC-1271 accepts `address(0)` recoveries** — impossible: the comparison is against `address(this)`, which is never zero.
- **High-s signature malleability in `_recover`** — deliberate mirror of raw `ecrecover` (what Permit2 does for EOAs); rejecting high-s would create the opposite simulation/reality divergence. Wallet signatures are canonical regardless.
- **10^50 sentinel detectable by malicious contracts** — inherent to forged-state simulation (injected code at the user's address is a bigger tell than the sentinel); randomization wouldn't remove detectability. Documented posture, not a fix.
- **Caller-supplied `tokenSlotOverrides` "let a dapp forge fake previews"** — trust-boundary confusion: `simulate()` args come from the wallet (the trusted caller), not the dapp.
- **Unbounded revert-reason string** — bounded in practice by RPC response caps and viem's decoder; not worth code.
- **Contract gas exhaustion under many candidates/probes** — speculative at 16M default with small real-world widths; noted as a maintenance concern only.
- **Layout-inference fallback untested** — it is tested: "falls back for non-standard allowance slots" pins 2 access-list probes and exact amounts.
- **Brittle debug-step-count assertions** — deliberate: pinning exact RPC counts is the library's performance regression net.
- **Mainnet tests skip silently without `MAINNET_RPC_URL`** — by-design opt-in.
- **`.editorconfig` missing** — marginal for a repo already formatter-enforced via oxfmt; skipped.
