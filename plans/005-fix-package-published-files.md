## Why this matters

The package manifest includes `docs/motivation.md` but not the image assets it references, so the published docs will have broken images. It also includes `src/generated/txSimulatorBytecode.ts`, which is source-only generated code that is already compiled into `dist`. The fix is just package hygiene: publish the built library, the contract source, and the complete motivation docs.

## Steps

### Step 1: Fix the `files` list

In `package.json`, replace the `files` list with:

```json
"files": [
  "dist",
  "contracts/TxSimulator.sol",
  "docs/motivation.md",
  "docs/assets/motivation"
]
```

Do not include `src/generated/txSimulatorBytecode.ts`; it is compiled into `dist/generated/txSimulatorBytecode.js` and `dist/generated/txSimulatorBytecode.d.ts`.

**Verify**: `node -e "const p=require('./package.json'); console.log(p.files.join('\\n'))"` -> prints exactly the four entries above.

### Step 2: Check the packed file list

Run:

```sh
pnpm pack --dry-run
```

Confirm the output includes:

- `docs/motivation.md`
- all seven files under `docs/assets/motivation/`
- `contracts/TxSimulator.sol`
- `dist/index.js`

Confirm the output does not include:

- `src/generated/txSimulatorBytecode.ts`
- `out/`
- `cache/`
- `node_modules/`

If `pnpm pack --dry-run` output is hard to inspect, use `pnpm pack --json --dry-run` and inspect the JSON file list manually. Do not add a new dependency.

**Verify**: dry-run exits 0 and the file list matches the bullets above.

### Step 3: Optionally extend the docs check only if useful

If the pack dry-run is easy to inspect manually, skip this step.

If you want a repeatable check, extend `scripts/check-docs.mjs` with a tiny package-files assertion by reading `package.json` and checking that `docs/assets/motivation` appears in `files`. Do not shell out from the script.

**Verify**: `pnpm check:docs` -> exit 0.

### Step 4: Run the final checks

Run:

```sh
pnpm check:docs
pnpm build
pnpm pack --dry-run
```

**Verify**: all three exit 0 and dry-run file list is correct.
