## Why this matters

The ghost contract's best-effort probes — `_tryBalanceOf` and `_tryAllowance` — issue uncapped `staticcall`s. Candidate addresses come from `eth_createAccessList` (any contract in the call path lands there) and balance queries can be caller-supplied, so a hostile or merely pathological `balanceOf` implementation burns 63/64 of the remaining gas per probe. Probes run once per candidate per call boundary, so one or two gas-burners OOG the entire simulation `eth_call` — which then surfaces as `StateOverrideUnsupportedError`, a *misclassified infrastructure failure*, instead of a per-token `unresolved`. The same vector poisons `balanceQueries.discoverErc20s` (one bad candidate kills the whole token filter) and `tokenOverrides.estimateRequirements` recon.

walletchan (the upstream implementation this library was inspired by) caps every probe at 100k gas and regression-tests it against a token whose `balanceOf` infinite-loops (see `docs/walletchan-learnings-2026-07-12.md` item 1). After this plan, a gas-burning token becomes `ok = false`: it is dropped from candidate observation and a balance query on it lands in `unresolved`, while every other probe still resolves.

```solidity
// contracts/TxSimulator.sol:248-254
function _tryBalanceOf(address token, address owner) internal view returns (bool ok, uint256 balance) {
    // forge-lint: disable-next-line(low-level-calls, calls-loop)

    if (!success || data.length < 32) return (ok, balance);
    ok = success;
    balance = abi.decode(data, (uint256));
}
```

## Steps

### Step 1: Add the gas cap to both probes

In `contracts/TxSimulator.sol`, add one constant next to the existing selector constants (after line 10):

```solidity
/// Gas forwarded to best-effort balance/allowance probes. Bounds hostile or pathological
/// implementations (e.g. a balanceOf that infinite-loops) to a fixed cost so one bad candidate
/// cannot OOG the whole simulation. 150k covers proxied tokens with hooks; walletchan ships 100k.
uint256 internal constant PROBE_GAS_LIMIT = 150_000;
```

Change both staticcalls to forward at most that much gas (the EVM forwards `min(PROBE_GAS_LIMIT, 63/64 of remaining)`, so this never reverts on low gas):

```solidity
(bool success, bytes memory data) = token.staticcall{gas: PROBE_GAS_LIMIT}(abi.encodeWithSelector(BALANCE_OF_SELECTOR, owner));
```

and equivalently in `_tryAllowance`. Keep the existing `forge-lint: disable-next-line` comments.

**Verify**: `pnpm build:contracts` → exit 0, and `git diff --stat src/generated/txSimulatorBytecode.ts` shows the file changed.

### Step 2: Create the gas-burner fixture

**Verify**: `pnpm build:contracts` → exit 0 (fixture compiles).

### Step 3: Regression test — balance query on a gas burner resolves as unresolved

In `test/simulate.test.ts`, add a test modeled on the existing unresolved-query cases (search the file for `unresolved`):

1. Deploy `TestToken` (mint the test account a balance, as existing tests do) and `GasBurner`.
2. `sim.simulate` with one real call (e.g. a small `TestToken.transfer`) and `balanceQueries` covering: `{ asset: "native", account: from }`, `{ asset: testToken, account: from }`, `{ asset: gasBurner, account: from }`.
3. Assert `status === "success"`, the TestToken delta matches the transfer, and the GasBurner query appears in the result's `unresolved` list (match the exact result shape used by the existing unresolved tests).

**Verify**: `pnpm test -- test/simulate.test.ts` → all pass, including the new test.

### Step 4: Behavioral test — a gas-burning candidate is filtered out of discovery

> **Amended 2026-07-12 during execution** (advisor decision, after the original pre-fix-failure requirement fired its own STOP condition): with `calls: []`, the discovery filter probes each candidate exactly once, and a single uncapped burner self-limits under the EVM 63/64 rule — the caller retains 1/64 of the budget, which suffices to finish. The compounding vector is checkpoint rounds in `simulate()`, which Step 3 proves (its test DOES fail pre-fix). This step is therefore a **behavioral pin**, not a discriminating regression: it need not fail pre-fix, and the corresponding STOP condition is withdrawn.

Add a second test:

1. Deploy `TestToken` and `GasBurner`.
2. Call `sim.balanceQueries.discoverErc20s` with two calls: a `TestToken.transfer` and `{ to: gasBurner, data: "0x", value: 0n }` (executing the burner makes its access-list probe revert, which discovery already normalizes; `call.to` is always kept as a candidate, so GasBurner reaches the token filter).
3. Assert the returned tokens include `TestToken` and do NOT include the GasBurner address.

**Verify**: `pnpm test -- test/simulate.test.ts` → all pass. (Step 3's pre-fix failure check remains mandatory and is the regression proof for this plan.)

## Maintenance notes

- **Deferred, deliberately**: returndata bombs. A malicious token can still return enormous returndata; high-level Solidity copies it into memory at the caller's (uncapped) expense. Fixing that class requires assembly probes with a fixed 32-byte output buffer — a follow-up if ever observed in the wild. The gas cap covers the demonstrated attack (compute burn).
- 150k is a knob: if a real proxied token's `balanceOf` legitimately exceeds it, the symptom will be that token showing up as `unresolved`/undiscovered. Bump the constant and regenerate rather than special-casing.
- Plan 047 also edits `contracts/TxSimulator.sol`; whichever executes second rebases a trivial diff. Run 044 first (this plan).
