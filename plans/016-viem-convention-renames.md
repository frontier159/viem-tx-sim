## Why this matters

Consumers are viem-native wallet developers. Two naming choices fight them:

1. `SimulatedCall.calldata` — viem transactions/calls and the ERC-5792
   `wallet_sendCalls` payload both use `data`. Today a `sendCalls` batch
   (`{to, data, value}[]`) cannot be passed to `simulate()` without mapping
   `data → calldata` on every call, on the hottest integration path.
2. The probed account is `from` in `simulate()`/`discoverRequirements()` but
   `owner` in `discoverBalanceSlots()`/`discoverAllowanceSlots()` — the
   README example literally writes `owner: from`. One concept, two names.

This package is pre-production (0.1.0): the rename is cheap now and expensive
after adoption. Everything else in this plan wave (017-022) is specified
against the new names, so this lands first.

## Current state

`src/types.ts:3-7`:

```ts
export type SimulatedCall = {
  to: Address;
  calldata: Hex;
  value?: bigint;
};
```

`src/slots.ts` — both public functions take `owner: Address`
(`discoverBalanceSlots` ~line 13, `discoverAllowanceSlots` ~line 39) and
forward it to internal probes as `owner: args.owner`. The internal probes'
own `owner` parameter name is **correct and stays** (it genuinely is the
mapping owner there; only the public arg names change).

Consumers of `call.calldata` inside src: `src/simulate.ts` (calls.map
normalization), `src/requirements.ts` (calls.map normalization,
`isAllowanceSetForSpender(call.calldata, ...)` via `firstInBatchAllowanceSetIndex`),
`src/internal/simulator.ts` (`data: call.calldata` when encoding),
`src/internal/discovery.ts` (`data: call.calldata` per access list).
Tests use `calldata:` keys throughout `test/*.test.ts`; README examples use
`calldata:`; CLAUDE.md may reference the field.

`src/requirements.ts` passes `owner: args.from` to
`discoverBalanceSlots` (public) — that call site simplifies to `from: args.from`.

Note: the word `calldata` also appears in `contracts/*.sol` as the Solidity
storage-location keyword — contracts are untouched; scope all greps to `.ts`
and `.md` files.

## Steps

### Step 1: `SimulatedCall.calldata` → `data`

Rename the field in `src/types.ts`, then fix every `.ts` usage (compiler
finds them): the two `calls.map` normalizations, `isAllowanceSetForSpender`'s
call site, simulator encoding, discovery access-list building, and every

**Verify**: `pnpm typecheck` → exit 0;
`grep -rn "calldata" src/*.ts src/internal/*.ts test/*.ts README.md` → no matches.

### Step 2: public `owner` → `from` in slots functions

In `src/slots.ts`, rename the public arg `owner` → `from` in both functions
(internally still forwarded as `owner: args.from` to probes). Update the
call site in `src/requirements.ts` and every test/README usage.

**Verify**: `pnpm typecheck` → exit 0;
`grep -n "owner" src/slots.ts` → only the forwarding lines
(`owner: args.from`), no public arg named `owner`;
`grep -n "owner: from" README.md` → no matches.

### Step 3: full gate

**Verify**: `pnpm verify` → exit 0. Test count unchanged (rename must not
add/remove behavior).
