## Why this matters

The current public method names center the probing mechanism (`discover*`) rather than what callers are trying to do. After plan 026, the slot discovery outputs are explicitly ready-to-use `TokenSlotOverride`s, so the API should say that. Rename the surface to: `prepareBalanceOverrides`, `prepareAllowanceOverrides`, and `estimateAssetRequirements`.

## Steps

### Step 1: Rename public types

In `src/types.ts`, rename:

```ts

BalanceSlotDiscovery           -> PreparedBalanceOverrides
DiscoverAllowanceSlotsArgs     -> PrepareAllowanceOverridesArgs

DiscoverRequirementsArgs       -> EstimateAssetRequirementsArgs
DiscoveredRequirements         -> EstimatedAssetRequirements
DiscoveredRequirementsSuccess  -> EstimatedAssetRequirementsSuccess
DiscoveredRequirementsReverted -> EstimatedAssetRequirementsReverted
```

Update JSDoc so it references the new method names. Keep result shapes unchanged.

**Verify**: `pnpm typecheck` may fail only on stale references to the old names.

### Step 2: Rename methods and internal bindings

In `src/txSimulator.ts`, rename interface methods and factory properties:

```ts
discoverBalanceSlots     -> prepareBalanceOverrides

discoverRequirements     -> estimateAssetRequirements
```

Rename internal implementation exports/imports in `src/internal/slots.ts` and `src/internal/requirements.ts` to match the public names. Keep file names as-is; the domains are still slots and requirements.

Update error messages that include old method names only if any exist. Do not change debug step names.
